## 10ステップでできること

### Step 1〜6：ヒアリングと組織設計

自治体名・都道府県・人口・主要課題を入力します。入力した情報はNotionに自動記録され、後からでも参照できます。

- 基本情報（自治体名・都道府県・地域特性）
- 主要課題の選択（人口減少 / 財政 / インフラ老朽化 / 医療 / 観光 など）
- 既存システムとの接続方針

### Step 7〜8：AI機能の選択

導入する AI 機能を選びます。現在は 10 種類のAIエンジンが選択肢として並んでいます。

| 機能名 | 対象課題 |
|---|---|
| LINE住民相談AI | 住民の困りごとを24時間受付 |
| 移住定着リスクAI | 移住者の転出を早期検知・防止 |
| 往診優先順位AI | 医師不足地域の訪問医療を最適化 |
| 施策PDCA | 施策の実行状況をリアルタイム管理 |
| 財政健全化AI | 財政指標の悪化を予兆検知 |
| インフラ老朽化AI | 橋梁・道路の更新優先順位を提示 |
| CO2削減トラッカー | 脱炭素進捗を自動集計 |
| 農業担い手マッチングAI | 農地と就農希望者をAIが結ぶ |
| 子育て流出リスクAI | 子育て世帯の転出懸念を検知 |
| 復興進捗ダッシュボード | 災害復興案件の進捗を一覧管理 |

---

📸 **【スクショ②】Step 7〜8 の機能選択画面**
> 推奨：チェックボックスで複数のAI機能が選べる画面。四万十市であれば「LINE住民相談AI」「移住定着リスクAI」「往診優先順位AI」「施策PDCA」の4つがONになっている状態。

---

### Step 9：テストデータの入力

各AI機能が正しく動くか確認するための初期データを入力します。たとえば移住定着リスクAIなら「移住者の氏名・移住時期・定住状況」を数件登録するだけ。このデータはNotionに自動保存されます。

### Step 10：ワンクリック自動構築 🚀

ここがハイライトです。

「AIページを自動構築する」ボタンを1回押すだけで、以下がすべて自動で実行されます。

1. **Notionに自治体ページを作成**（「🏙️ 自治体・組織 展開ページ」配下）
2. **選択した機能ごとのDB（データベース）をNotionに自動作成**
3. **アプリのコード（municipalities.ts・municipality-db-config.ts）を自動生成**
4. **サイドバーへの表示設定を更新**

つまり、プログラマーが手でやっていた「DB設計→Notion構築→コード修正→デプロイ」という4工程が、ボタン1回で完了します。

---

📸 **【スクショ③】Step 10 の完了画面**
> 推奨：「✅ 自動構築が完了しました」というメッセージと、作成されたNotionページへのリンクが表示されている画面。できればNotionページのURLも見えているとベスト。

## なぜこれが重要なのか

日本には1718の市区町村があります。

これまで「AIで自治体DXを支援する」というサービスは、**1つの自治体に1つのシステムを作る**という形がほとんどでした。つまり、自治体の数だけ開発コストと時間がかかる。

RunWith Platform が目指しているのは、その構造を変えることです。

1つのプラットフォームで複数の自治体を安全に運用する「マルチテナント設計」を採用し、ウィザードで自治体ごとの設定を自動生成することで、**横展開のコストを限りなくゼロに近づける**。

今回のウィザード完成で、その設計がひとつの形になりました。
