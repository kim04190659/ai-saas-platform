# 自治体DXを「誰でも使えるもの」にする——RunWith Platform 最新8スプリントの記録

> Sprint #71〜#78（2026年4月〜5月）の開発レポートです。

---

## はじめに

「AIで自治体を変える」と言いながら、実際に職員の方が毎日使えるツールになるには、まだまだ乗り越えるべき壁がある——そう痛感しながら開発を続けています。

今回の8スプリントは、**「機能を増やす」から「使いやすく整える」への転換点**でした。  
新しい自治体を1時間以内に追加できる仕組み、左メニューの全面刷新、AIアシスタントの全ページ統一。  
地味に見えますが、現場に届けるためにはここが一番大事だと思っています。

---

## Sprint #71・#72：自治体追加ウィザード 2.0

### 「ウィザードを進めるだけで新しい自治体が動く」を目指して

RunWith Platform は現在、霧島市・屋久島町・海士町・五島市・輪島市・西粟倉村・上勝町・神埼市・気仙沼市・四万十市の10自治体に対応しています。  
ただし、これまでは新しい自治体を追加するたびに、開発者がコードを手書きして設定ファイルを更新する必要がありました。

Sprint #71では、この作業を**ウィザード画面から進めるだけで完結させる**ための土台を作りました。

**追加したステップ**

- **Step 7：基本機能確認**  
  RunWith が提供する4つの必須機能（WBダッシュボード・AIアドバイス・職員サポート・LINE相談）を自治体担当者が確認するステップ。「この機能は使いますか？」をGUIで確認できます。

- **Step 8：拡張AI機能の選択**  
  350件以上の機能カタログ（`feature-catalog.ts`）から、自治体の課題に合った拡張AI機能を選択するステップ。廃棄物最適化・インフラ老朽化・財政健全化・観光管理・農業担い手マッチング……など、これまで開発してきたAIエンジンを一覧から選べるようにしました。

- **Step 9：テストデータ入力**  
  選択した機能に対応するサンプルデータを、ウィザード画面から自動でNotionに投入するステップ。初日から「動いている感」を体験してもらえます。

## Sprint #73：Notion自動プロビジョニング

### ボタン1つで、Notion DBが丸ごと構築される

Sprint #73の目玉は、**「自動構築を開始」ボタン**です。

ウィザードの最終ステップで自治体ID・短縮名・テーマカラーを入力してボタンを押すと——

1. 選択した機能に必要なNotion DBを一括自動作成
2. `municipalities.ts`（自治体マスタ）と `municipality-db-config.ts`（DB IDマッピング）に追加すべきコードを自動生成してコピーボタンで提示
3. 完了画面でNotionページへのリンクを表示

これにより、**新しい自治体の追加作業が数日→1〜2時間**に短縮されました。  
実証として、このフローで四万十市（高知県）を追加しました。

```
「コードを書かずにNotionに自治体の環境ができる」
これが実現したことで、将来的には自治体担当者自身がセットアップできる可能性が見えてきました。
```

## Sprint #74：メニューと全自治体ダッシュボードの整備

### 「何ができるのか」が一目でわかるメニューへ

機能が増えるにつれて、左メニューが複雑になりすぎていました。  
Sprint #74では、メニューを**「必須4機能 + 基本AIセット + 課題特化型AI」**の3層構造に再設計しました。

同時に、全10自治体のWell-Beingダッシュボードを統一フォーマットで整備。  
Notionに**21本のDB（7自治体×3DB）**を作成し、それぞれ63件のサンプルレコードを投入しました。  
どの自治体を選んでも同じUIで住民WBスコア・施策進捗・課題リストが確認できる状態になりました。

## Sprint #75：サイドバーリンクで自治体を自動切り替え

### URLパラメータを使った「どこからでも自治体を切り替えられる」設計

これまでは、ヘッダーのドロップダウンで自治体を切り替えてからページを開く必要がありました。  
Sprint #75では、サイドバーの各リンクに `?municipalityId=xxx` パラメータを付与し、  
**リンクをクリックするだけで自治体とページが同時に切り替わる**ようにしました。

技術的には `MunicipalitySearchParamsSync` という新しいコンポーネントを作り、  
URLパラメータをReact Contextに自動同期する仕組みを導入しました。  
`Suspense` でラップすることで、Next.jsのサーバー・クライアント両対応も実現しています。

## Sprint #76：左メニュー全面リデザイン

### 「縦に長すぎる」問題を解消

職員の方にデモをすると、「メニューが多すぎて何を押せばいいかわからない」という声をよくいただきました。  
Sprint #76でサイドバーを全面リデザインしました。主な変更点は4つです。

**① 空のモジュールを非表示にする**  
その自治体でまだ使っていない機能は、メニュー自体を表示しないようにしました。  
「準備中」のリストが並んでいると、何が使えるのかわかりにくいためです。

**② 必須4機能は常時展開**  
毎日使う4つのコア機能（ダッシュボード・AIアドバイス・職員サポート・LINE相談）は、  
開閉トグルなしで常に表示されます。「全体像が一目でわかる」状態を作りました。

**③ AI拡張・自治体管理・運用管理はデフォルト閉じる**  
普段は閉じておき、必要なときだけ開く。縦方向の長さを大幅に削減しました。

**④ テキスト折り返しを修正**  
機能名が長いとき、バッジと文字が重なって見づらかった問題を解消しました。

## Sprint #77：AIアシスタントパネルを全ページで統一

### 「ページごとに違う名前・違う内容」を解消する

RunWith Platform には「RunWithアシスタント」という右パネルがあります。  
職員の方がわからないことをAIに聞ける機能ですが、ページによって名前がバラバラでした。

- あるページでは「RunWithアシスタント」
- あるページでは「屋久島AIアドバイザー」
- あるページでは「霧島市AIアドバイザー」

内容も「業務アドバイス」だったり「使い方ガイド」だったり、一貫していませんでした。

**Sprint #77では、ChatPanelを全面的に作り直しました。**

`usePathname()` でユーザーが現在いるページを自動検出し、  
そのページ固有のコンテキスト（説明・システムプロンプト・よくある質問）を表示します。  
60以上のページに対応したコンテキストマップを実装しました。

名称は全ページで**「RunWithアシスタント」**に統一。  
それまで霧島市専用として存在していた `KirishimaChatPanel` を廃止し、共通の `ChatPanel` に統合しました。

```
どのページを開いても、右上の「RunWithアシスタント」を押せば
そのページの使い方を教えてくれる——という体験を実現しました。
```

## Sprint #78：研修・学習ページにもAIアシスタントを追加

### カードゲームの使い方を、ゲーム中に聞ける

RunWith Platform には、ITIL/SIAMの知識をゲームで体験できる「PBLカードゲーム」や、  
縮退する自治体を舞台にした「Well-Being QUEST」などの研修コンテンツがあります。

これらのページはゲームのフルスクリーン体験を優先した設計のため、  
通常のダッシュボードとはレイアウトが異なります。Sprint #77で作ったダッシュボード用のレイアウトはそのままでは使えませんでした。

**Sprint #78では、ゲームのUIを壊さず右パネルを追加する方法を実装しました。**

- 画面右下に固定表示される「使い方ガイド」ボタン（フローティングボタン）
- ボタンを押すと、ゲーム画面の上にパネルがオーバーレイ表示される
- パネルにはそのゲーム固有の説明・操作方法・よくある質問が表示される

Next.jsの `layout.tsx` を活用し、`card-game/` と `well-being-quest/` 以下の全ページに一括適用しました。  
ゲームのフルスクリーン体験を守りながら、いつでもヘルプを呼び出せる設計です。

## この8スプリントで見えてきたこと

今回の開発を通じて、**「機能の数」よりも「整えること」の重要性**を改めて実感しました。

自治体の職員の方は、新しいシステムに慣れるだけでも時間がかかります。  
「メニューが多すぎる」「どこを押せばいいかわからない」「名前がバラバラで混乱する」——  
こういった小さなストレスが積み重なって、使われないシステムになってしまいます。

RunWith Platform が「継続して使われるツール」になるために、  
今はUIの整備・操作の一貫性・ヘルプの充実に力を入れています。

**次のスプリントでは、実際に職員の方にデモをして、フィードバックを集める予定です。**  
「動くものを届けて、一緒に育てる」——このサイクルを早めていきます。
